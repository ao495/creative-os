# 創造OS (Creative OS) 🚀

[![Version](https://img.shields.io/badge/version-1.1-blue.svg)](https://github.com/your-repo/creative-os)
[![License](https://img.shields.io/badge/license-MIT-green.svg)](LICENSE)

**nostall開発から抽出された創造の理論体系**

創造OSは、nostall開発ログから抽出された創造構造を基に、**創造の発生メカニズム**を体系的にモデル化した理論フレームワークです。

従来のAIシステムが「情報処理」や「推論」を扱うのに対し、創造OSは**「創造そのものの構造」**を扱います。

---

## 🌟 特徴

- **構造的創造**: 点ではなく「構造」として創造を扱う
- **重力場モデル**: 価値が質量を持ち、創造を引き寄せる
- **進化論的**: 創造は9段階で進化する
- **回路構造**: 創造には「神経回路」のような連鎖構造がある
- **宇宙モデル**: 創造空間を3D宇宙として可視化

## 概要

創造OSは、以下の10個の創造ベクトルMiniから構成されます：

1. **Core抽象Mini** - 本質線の抽出
2. **意味干渉Mini** - 干渉波の解析
3. **未来収束Mini** - 未来方向の予測
4. **意味重力Mini** - 重力井戸の形成
5. **創造地図Mini** - 創造地形の可視化
6. **本質ルール抽出Mini** - 法則の抽出
7. **目的ベクトルMini** - 創造方向の定義
8. **干渉波純度Mini** - 価値とノイズの分離
9. **本質圧縮Mini** - 本質式の生成
10. **意味連鎖Mini** - 創造連鎖の生成

これらに加えて、**LLM補助モジュール**が自然言語レイヤーを提供します。

### 実データから抽出された構造

- **本質線**: 44本
- **重力井戸**: 5個
- **本質ルール**: 65個
- **本質式**: 3個
- **意味連鎖**: 4本

## 創造進化論（9段階モデル）

創造OSは以下の9段階で進化します：

> **静止 → 運動 → 構造 → 宇宙 → 法則 → 意図 → 精度 → 速度 → 生命**

### 進化の方向性

1. **静的 → 動的**: 点から流れへ
2. **無秩序 → 構造**: 混沌から秩序へ
3. **不可視 → 可視**: 暗黒から地図へ
4. **暗黙 → 明示**: 経験から法則へ
5. **受動 → 能動**: 反応から方向性へ
6. **低速 → 高速**: 探索から発明へ
7. **部分 → 全体**: 機能から完全体へ
8. **機械 → 生命**: システムから話せる存在へ

**詳細**: [創造進化論レポート](creative_evolution_theory.md)

## 創造OSの構成要素

### 1. 重力井戸モデル

価値の高いテーマは「質量」を持ち、創造のエネルギーを引き寄せる重力井戸を形成します。

- 質量（価値強度）が大きいほど、創造を強く引き寄せる
- 意味空間における位置が価値の文脈を決定
- 高安定性の重力井戸は長期にわたって価値を維持

**可視化**: [重力井戸マップ](visualizations/gravity_wells_map.png) | [3D創造惑星モデル](planet_model_3d/creative_planet_model_3d.html)

### 2. 本質線（Essence Lines）

本質線は創造の「構造の軸」であり、複雑な構造から本質を「線」として抽出したもの。

- **構造性**: 創造の「骨格」を形成
- **座標性**: 意味空間における「座標軸」として機能
- **重力指向性**: 本質線は重力井戸の方向に向かう

### 3. 意味連鎖（Semantic Chains）

意味連鎖は創造の「思考の流れ」であり、異なる創造要素を接続して「創造の動線」を生成します。

### 4. 創造回路図

創造OSには神経回路のような連鎖構造があり、原理公式と因果パターンから「創造の回路図」が構築されます。

- **進化チェーン（赤線）**: 創造OSの進化段階を示す
- **因果関係（青線）**: 条件と結果の関係。創造の発火パターン
- **関連関係（グレー線）**: 概念間の弱い接続。潜在的な創造の連鎖

**可視化**: [創造回路図](circuit_analysis/creative_circuit.png)

### 5. 創造惑星モデル（3D創造宇宙）

創造OSの構造は3次元空間で可視化でき、重力井戸が惑星、本質線が軌道、意味連鎖が宇宙航路として表現されます。

**可視化**: 
- [インタラクティブ3D宇宙モデル](planet_model_3d/creative_planet_model_3d.html)
- [静的3D画像](planet_model_3d/creative_planet_model_3d.png)

## 主要な概念

### 本質式

複雑な創造構造を一行の式に圧縮したもの。圧縮率が高いほど、より本質的な要素を抽出できていることを示します。

**例**: 「価値」（圧縮率 99.33%）

### 原理公式

nostall開発における「核心的な法則」。14個の原理公式が抽出されています。

**詳細**: [本質式とルールの分析レポート](essence_and_rules_analysis.md)

### 因果パターン

条件と結果の関係を示すパターン。42個の因果パターンが抽出されています。

| パターン例 | 条件 | 結果 | 信頼度 |
|-----------|------|------|--------|
| structureタイプの本質線は複数の本質線と接続される傾向がある | 本質タイプ: structure | 構造の形成, 関係性の確立 | 0.456 |

## 用途と適用例

創造OSは、以下の用途に適用できます：

### 1. LLMの創造的探索エンジン

LLMの出力から本質線を抽出し、創造の構造を可視化することで、LLMの「思考過程」を理解・最適化できます。

### 2. 研究開発の発見支援

研究ログや開発履歴を創造OSで解析することで、「価値のある方向性」を自動的に発見できます。

### 3. 企業内の意思決定補助AI

会議議事録や意思決定ログを創造OSで処理することで、企業の「意思決定パターン」を可視化し、最適化できます。

### 4. 新規事業の探索と評価

複数のアイデアを創造OSで解析することで、「最も価値の高い方向性」を重力井戸として特定できます。

### 5. 人間のクリエイティブ増幅

個人の思考ログや創作過程を創造OSで解析することで、「自分の創造パターン」を理解し、創造力を増幅できます。

### 6. LLMの内部可視化技術（I-XR）との統合応用

I-XRによる「推論構造の可視化」と創造OSによる「創造構造の可視化」を組み合わせることで、LLMの「内部プロセス」を完全に可視化・最適化できます。

## なぜ創造OSが発見を加速するのか？

1. **構造の可視化**: 創造の「地形」を可視化し、どこに価値があるかが明確になる
2. **重力指向の探索**: 重力井戸は「価値の中心」を示し、価値の高い領域に自然に集中できる
3. **進化の予測**: 現在の段階から次の段階を予測でき、最適な介入ポイントが明確になる
4. **回路構造の活用**: 「どこに創造のスイッチがあるか」が分かり、創造を意図的に発火させられる
5. **法則の再利用**: 抽出された法則は新しい創造にも適用可能で、創造の再現性が向上する

## nostallとの統合

創造OSは、nostallシステムの「創造エンジン」として機能します。

### nostallにおける創造OSの役割

1. **判断の補助**: Stop OSによる判断を、創造的観点から補完する
2. **発見の加速**: I-XRによる解析結果から、創造的な発見を加速する
3. **意思決定の強化**: 目的ベクトルにより、創造的な意思決定を支援する
4. **学習の構造化**: 経験から法則を抽出し、学習を構造化する

### 統合コンポーネント

- **Commander**: 創造OSの結果を判断材料として活用
- **Mini Agent**: 創造ベクトルMiniとして実装
- **Factory Dashboard**: 創造地図を可視化
- **I-XR**: 解析結果を創造OSの入力として活用

```bash
# nostallログのバッチ処理
python nostall/minis/creative_vector_minis/run_nostall_logs_batch.py \

    --max-files 1
```
